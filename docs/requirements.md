### プロダクト概要

- **目的**: RAW初心者〜中級者向けの「大量の写真をサクサク選別し、そこそこ高画質で現像できる」デスクトップアプリ。
- **特徴**:
  - フィルムストリップ＋メイン画像でテンポよく写真を送れる。
  - 編集は非破壊（RAWはそのまま、編集情報は別管理）。
  - 将来的にAIを深く統合する「AIネイティブ」な方向性（MVPではAIなし）。

関連: `docs/external-design.md`（外部設計 v0.1） / `docs/internal-design.md`（内部設計 v0.1）

### ターゲット / ユースケース

- **ターゲットユーザー**: RAW現像に慣れ始めた初心者〜中級者。
- **主なユースケース**:
  - 撮影後に1フォルダ分のRAWを読み込む。
  - ★レーティングで選別しつつ、必要な写真を軽く調整する。
  - 仕上がった写真を sRGB JPEG で書き出す。

### プラットフォーム / 技術スタック（想定）

- **プラットフォーム**: macOS デスクトップ（MVPはmacOS限定）。
- **技術スタック（第一候補）**:
  - バックエンド: Rust
  - RAW処理: LibRaw（RustからFFI）
  - デスクトップシェル: Tauri
  - フロントエンド: React + TypeScript
- **AI連携（将来）**:
  - OpenAI等のクラウドAPIを利用予定（MVPではAI機能なし）。

### 対応フォーマット（MVP確定）

- **入力（RAW）**: **Sony `.arw` のみ**（MVP）
- **出力**:
  - **sRGB JPEG**（MVP。解像度は元解像度固定）

### 機能要件（MVP範囲）

#### Must

- **フォルダ読み込み**
  - 指定フォルダ内のRAWファイル一覧を読み込む。
- **フィルムストリップ**
  - 画面下部または側面に横一列サムネイルを表示。
  - サムネクリックでメイン画像を切り替える。
- **メイン画像表示**
  - 選択したRAWのプレビュー表示。
  - ズーム / パンに対応。
- **ヒストリーパネル**
  - スライダー1つ動かすたびに1ステップとして履歴を追加。
  - 過去ステップへの戻り（Undo的動作）ができる。
- **編集パラメータ**
  - 以下のスライダーを右パネルに配置:
    - 露出
    - コントラスト
    - ハイライト
    - シャドウ
    - 黒レベル
    - 白レベル
    - 彩度
    - 自然な彩度
    - 色温度
- **プレビュー性能**
  - スライダー操作後、3〜5秒程度以内にプレビューが更新されれば許容。
  - 画質優先（多少重くてもよい）。
- **書き出し**
  - 現在選択中の写真を **sRGB JPEG** で1枚書き出す。
  - 画質プリセット: **高画質 / 普通 / 低画質** を選択できる（MVP）
  - 解像度: **元解像度固定**（MVP。将来変更可能）
  - メタデータ: **MVPは付与しない（または最小限）**。まずは安全に「なし」で進める（詳細は外部/内部設計で確定）

#### Want（MVPで入ると嬉しい）

- **★レーティング**
  - 0〜5段階のレーティング。
  - フィルムストリップやメイン画像付近から設定可能。
- **レーティングの保存**
  - RAWファイルとは別に、サイドカー形式で保存（他ツールとの互換は必須ではなく、自アプリ内で完結していればよい）。
- **Before/After**
  - 現在の編集状態と「編集なし（デフォルト現像）」を切り替え表示。

#### Later（MVP後）

- 書き出し形式の選択（JPEG/PNG/TIFFなど）。
- グリッドビュー（Lightroom風の一覧選別）。

### データ / 保存に関する要件

- **互換性ポリシー**
  - Lightroom / Capture One との完全互換は重視しない。
  - 自アプリ内で完結していればよい。
- **保存形式の方針**
  - RAWは常に非破壊。
  - レーティングや編集パラメータ:
    - 外部サイドカー（XMPライクまたは独自フォーマット）として保存。
    - 将来AI情報や高度な履歴を持つ場合は、別ファイルまたはDBで拡張する。

### 色管理（MVP方針）

- **プレビューと書き出しの見た目を一致**させることを優先する。
- **MVPは常に sRGB に変換してプレビュー/書き出し**する（将来拡張）。

### UI / UX 要件

- **レイアウト（1画面構成）**
  - 上または左: フォルダ選択。
  - 下（または左右どこか）: フィルムストリップ（横スクロールサムネ）。
  - 中央: メイン画像（ズーム / パン）。
  - 右: 編集スライダー＋ヒストリーパネル。
- **操作性**
  - MVP段階ではマウス操作メイン。
  - キーボードショートカットは後回し（将来追加）。
- **初心者向け配慮**
  - 各スライダーや主要ボタンにツールチップや簡潔な説明テキストを付与し、「これ何？」がすぐ分かるようにする。
  - パラメータ数は上記の範囲に抑え、まず「迷わせない」ことを重視する。

### 非機能要件

- **パフォーマンス**
  - 1フォルダに数十〜数百枚程度を想定（正確な上限は別途調整）。
  - スライダー1操作あたりのプレビュー更新時間: 3〜5秒以内（MVPとして許容）。
- **品質**
  - プレビュー画質は「そこそこちゃんとした画質」を優先。
  - 必要であれば将来、「高速プレビュー」と「高品質プレビュー」の二段構えも検討する。

### 開発/計測の前提（MVP）

- **基準マシン**: M2 Mac
- **基準データセット**: `data/medium`
- **計測指標（最低限）**:
  - フォルダ選択 → フィルムストリップ表示まで（ロード体感）
  - サムネ表示（先頭N枚が出るまで。例: N=20）
  - Dragging（操作中）: 追随の遅延
  - Commit（操作確定）: 高品質プレビューへの置換まで（目標: 3〜5秒）
